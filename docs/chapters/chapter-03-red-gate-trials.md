# Chapter 03 — Red Gate Trials

## Purpose

Teach that assumptions kill. Before a squad commits to a major move, the player must prove the tactic works.

## Story summary

A red-class gate traps the player’s raid party in a hostile environment where the obvious path keeps failing. Veteran instincts break down. The player gains respect by introducing trial runs, decoys, and verification rituals before high-risk actions. Competence begins to replace desperation.

## Learning goal

Test-Driven Development + Reason.

## Scenes

### red_gate_entry
The gate seals behind the team.

### frozen_ambush
Enemy behavior contradicts the guild briefing.

### proof_room
The player discovers a chamber where tactics can be simulated at a cost.

### squad_conflict
Han Doyun resists “overthinking” until brute force backfires.

### verified_assault
The player launches a plan only after passing enough proof checks.

## Choices

- Run a risky live attack immediately.
- Spend time validating the route.
- Split the party for faster coverage.
- Preserve resources for a longer but safer clear.

## Win state

The player defeats the gate commander using a tested plan and earns visible squad respect.

## Fail state

The player clears the gate but with severe losses, harming later chapters.

## Rewards

- Tactics validation system unlocked
- Reputation tag: “Planner”
- Better team command options
